8-Puzzle A* Manhattan

Inteligencia Artificial, Primavera 2017
Alejandro Almora 149634, Miguel Velázquez 149402

Pequeño programa para resolver 8-puzzles. Utilizando Common Lisp (CCL) y Java para el GUI.

El programa utiliza el algoritmo A*, para realizar busquedas. Usamos Una funcion Heuristica Costo(nodo) = Distancia_Manhattan(nodo) + nivel(nodo)

Tambien se programaron las siguientes funciones

  • Element_At: Regresa el elemento en la posicion index (0 a Length - 1)
  • FirstN: Regresa los primeros N elementos de la lista
  • Manhattan: Funcion heuristica
  • PosX: Regresa la coordenada X (Matriz inducida)
  • PosY: Regresa la coordenada Y
  • Swap: Intercambia dos nodos
  • GetZero: Encuentra la posicion del 0 (Libre)
  • Up,Down,Right,Left: Funcion que hace movimientos a los lugares indicados, regresa NIL si no es un movimiento valido.
  • MakeMove: hace el movimiento indicado
  • Expand: Expande un nodo, regresa la lista de hijos formados
  • CalculaCosto: Función que calcula el valor de la heurística
  • IsInClosedStack: funcion que verifica si un nodo se encuentra en cerrado.
  • Backtrack: reconstruye el camino seguido para la solución
  • Print_Puzzle: Auxiliar para ver movimientos en Lisp
  • ADDOPEN|ADDCLOSE: Agrega a Abierto o Cerrado un nodo. En abierto busca su lugar.
  • Path: funcion que hace la busqueda

Se utilizó Java para la realizacion de la GUI, con Java usando el Runtime actual, mandabamos un thread para correr el binario de CCL, el thread de Java lo ponemos a dormir por 5,000ms para darle tiempo a CCL de ejecutarse. Lo malo de llamar un Thread para el proceso de CCL es que no hay una forma de ver cuando existe un error y por lo tanto no podemos mandar algun error.

El programa no es eficiente en el numero de movimientos

Para la ejecucion de Java se necesita editar la siguiente funcion, ya que requiere rutas absolutas.

Linea de codigo: 471 del Archivo MainGUI.java
Runtime.getRuntime().exec("/usr/local/bin/ccl64 --load /Users/alxalmora/Documents/8puzzle/Lisp/src/8puzzle.lisp");
